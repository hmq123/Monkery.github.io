搭建属于自己的博客——Github+Hexo

搭建指南

百度一下，关于Github+Hexo搭建博客的相关资料很多，我也是在前人的基础上学习，希望能帮助大家快速搭建博客。

一、环境准备

1、Node.js安装

安装教程先自行百度，后期有时间奉上。直接进入验证node.js是否正确安装：打开cmd，输入以下命令：

node -v

npm -v

如下图出现版本号信息即已成功安装：

2、Git安装与配置

和Node.js一样，大部分设置保持默认就行了，下面检查Git是否安装正确，打开命令行输入：

git --version

如果结果如下图所示，则说明安装正确，可以进行下一步了，如果不正确，则需要回头检查自己的安装过程。

二、安装hexo

1、创建hexo文件夹

在合适的地方创建hexo文件夹，这里我创建在F:/hexo

2、打开cmd，进入hexo，输入以下命令行：

npm install hexo-cli -g

npm install hexo --save

3、检查hexo是否安装正确

hexo -v

三、Hexo配置

1、初始化Hexo

首先要确保F:/hexo文件夹为空文件夹，然后继续上面的操作，在命令行中输入：

hexo init

2、安装组件

在命令行输入：

npm install

3、生成hexo

在命令行输入：

hexo g

4、预览hexo

在命令行输入：

hexo s 会提示：



然后浏览器打开http://localhost:4000/，你将会看到：



PS:hexo默认端口为4000，若hexo安装成功，可以正常运行，启动也没问题，但是不能访问，那可能是4000端口被占用了。这时，你在启动hexo的时候在命令行输入hexo s -p 5000换一下端口便能正常访问了。

四、修改配置文件

1、注册Github账号

2、创建Repository

创建的时候注意Repository的名字。比如我的Github账号是hmq123，那么我应该创建的Repository的名字是：hmq123.github.io。

3、修改配置文件

1)拷贝Repository地址

2)编辑_config.yml文件（在F:\hexo下）。

3)修改文件里面的deploy节点。其中的repository就改成你刚刚复制的地址，最后保存这个文件。

五、设置SSH keys

1、在Git Bash输入以下指令（任意位置点击鼠标右键），检查是否已经存在了SSH keys。

ls -al ~/.ssh



上图表示不存在，不存在进入第二步。下图表示存在SSH Keys:



如果存在的话直接删除.ssh文件夹里面的所有文件。

2、输入以下指令（邮箱就是你注册Github时候的邮箱）后，回车：

ssh-keygen -t rsa -C "380***0@qq.com"



然后它会提示要你输入passphrase（如上图，我没有输入直接回车，如果你输入的话，要记得，到时候会用到）。

3、然后键入以下指令：

ssh-agent -s

4、继续输入指令：

ssh-add ~/.ssh/id_rsa

输入之后，在我这里是出错了，不知道你的有没有出错。



如果你的也是这样子出错了的话，就输入以下指令：

eval 'ssh-agent -s'

ssh-add

5、复制key

键入以下指令，拷贝Key（先拷贝了，等一下可以直接粘贴）：

clip < ~/.ssh/id_rsa.pub
